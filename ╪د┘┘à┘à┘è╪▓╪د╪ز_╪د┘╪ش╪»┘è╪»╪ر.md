# مدد — المميزات الجديدة (مُبرمَجة ومُختبَرة)

تمت برمجة 4 مميزات + بيانات تجريبية، واجتاز المشروع **فحص TypeScript والبناء الإنتاجي** بنجاح.

## ما الذي أُضيف؟
1. **🗺️ فلترة بالمدينة** — قائمة مدن في صفحة البحث؛ تعرض موردي المدينة المختارة وتحسب
   الأسعار (أرخص/متوسط/أعلى) من عروضهم فقط. تُحفظ في الرابط `?city=`.
2. **⭐ المفضّلة** — زر قلب على بطاقات المنتجات وصفحة المنتج، وصفحة `/favorites`، ورابط
   في الشريط العلوي (للمستخدم المسجّل). الضغط بدون تسجيل دخول يوجّه لصفحة الدخول.
3. **🖼️ صور المنتجات** — رفع صورة عند إضافة المنتج من لوحة المورّد (Supabase Storage)،
   وتُعرض في البطاقات وصفحة المنتج مع رجوع للأيقونة عند عدم وجود صورة.
4. **🛒 طلبات عروض الأسعار (RFQ)** — زر "اطلب عرض سعر" على كل عرض، نافذة كمية+ملاحظة،
   صفحة "طلباتي" للمشتري (`/my-requests`)، وقسم "طلبات واردة" في لوحة المورّد للرد بسعر/حالة.

كما أُصلح عرض اسم الفئة في صفحة المنتج، وبقي الاسم "مدد" كما هو.

## خطوة مهمة: شغّل الـ migrations على قاعدة البيانات
الكود يعتمد جداول/أعمدة جديدة. طبّق الملفين (بالترتيب) من مجلد `supabase/migrations/`:
1. `20260608140000_seed_demo_data.sql` — البيانات التجريبية.
2. `20260608150000_favorites_images_quotes.sql` — جداول المفضّلة والطلبات + عمود الصورة + مخزن الصور.

**الطريقة:** Supabase → SQL Editor → الصق محتوى كل ملف → Run. (أو عبر Supabase CLI: `supabase db push`).

## تشغيل المشروع محلياً
```bash
npm install --legacy-peer-deps
npm run dev      # يولّد المسارات ويشغّل الموقع
```
> ملاحظة: `--legacy-peer-deps` بسبب تعارض إصدار nitro في تبعيات Lovable (لا يؤثر على العمل).
